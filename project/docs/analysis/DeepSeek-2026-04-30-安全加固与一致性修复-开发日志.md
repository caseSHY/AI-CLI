### 1. 沙箱安全加固

#### 1.1 问题根因

`require_inside_cwd()` 是一个已有安全原语，可在目标路径不在当前工作目录内时抛出 `unsafe_operation` 错误。但在本轮之前，以下命令未调用该检查：

| 命令 | 缺陷描述 |
|---|---|
| `rm`（非递归文件） | 仅对 `--recursive` 目录调用了 `require_inside_cwd`，非递归文件删除无边界检查 |
| `tee` | 完全未对输出路径做 cwd 边界校验 |
| `truncate` | 完全未对目标路径做 cwd 边界校验 |
| `install` / `ginstall` | 目标路径无 cwd 边界校验 |
| `dd` | 输出路径无 cwd 边界校验 |

#### 1.2 修复方案

统一为上述命令在执行前增加 `require_inside_cwd(path, cwd, allow_outside_cwd=False)` 调用。

涉及文件：`src/agentutils/fs_commands.py`

| 命令 | 修改点 | 新增代码 |
|---|---|---|
| `command_rm` | 将 `require_inside_cwd` 从仅 `path.is_dir()` 条件下移至所有路径 | 无条件对所有目标路径校验 |
| `command_tee` | 函数体开头新增 `cwd` 获取和遍历校验 | `require_inside_cwd(path, cwd, allow_outside_cwd=False)` |
| `command_truncate` | 函数体开头新增 `cwd` 获取和遍历校验 | `require_inside_cwd(path, cwd, allow_outside_cwd=False)` |
| `command_install` | 文件安装分支新增 cwd 校验 | `require_inside_cwd(destination, cwd, allow_outside_cwd=False)` |
| `command_dd` | 输出路径分支新增 cwd 校验 | `require_inside_cwd(output_path, cwd, allow_outside_cwd=False)` |

#### 1.3 安全边界说明

修复后，以下逃逸向量均被阻止：

- 相对路径遍历：`../outside/file.txt` → 拒绝（exit 8）
- 绝对路径：`C:\outside\file.txt` → 拒绝（exit 8）
- 符号链接指向 cwd 外部：`resolve_path` 已解析真实路径，sandbox 基于解析后路径校验 → 拒绝（exit 8）

### Summary

This development log covers three areas of improvement:

1. **P0 sandbox hardening** — closed 5 known escape gaps.
2. **Catalog/registry consistency** — aligned command priorities between `catalog.py` and `registry.py`.
3. **Test activation** — removed `@unittest.skip` decorators from now-fixed gap tests.

### Changes

- **Consistency**: Added missing commands to `catalog.py` (`mknod`, `ginstall`, `arch`, `hash`) and `registry.py` (`hash`); moved `shred` from P1 to P3 in `registry.py`.
- **Tests**: Removed 6 `@unittest.skip` decorators from `test_sandbox_escape_hardening.py`; updated module docstring.
